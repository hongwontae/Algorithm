1. 알고리즘이란?
=> 특정 작업을 달성하기 위한 과정이나 일련의 단계을 의미한다.


2. 알고리즘 문제를 해결하기 위한 방법
2-1. 계획 수립
2-2. 일반적인 문제 해결 패턴 파악


3. 2-1의 절차

3-1. 문제의 이해
=> 질문을 받으면 자신만의 방식으로 이해해야 한다.
=> 인풋과 내가 생각해야할, 풀어야할 output을 생각하자.

3-2. 구체적인 예제
=> 문제를 이해했다면 구체적인 풀이 예시를 떠올려보자.

3-3. 세부 분석
=> 문제를 해결하기 위해서 단계적으로 해결 방법을 제시한다.
=> 객체를 반환해야 하는가? 배열? 원시타입?
=> 그렇다면 객체를 먼저 선언할까? for/forEach/Map 어떤 메서드를 구현할지 판단
=> 규칙에 의거하여 어떤 값을 제거하거나 변형하기 위해 if문 분기를 어떻게 할 것인가?

3-4. 해결 or 단순화
=> 단순화하여 문제를 해결하여 어려운 부분을 이해한다.

3-5. Refactoring & Look Back
=> 다른 해결방법을 도출할 수 있는가?
=> 단번에 이해할 수 있는 코드인가?
=> 너의 코드를 향상시킬 수 있는가?
=> 그렇다면 리팩토링할 수 있는가?
=> 다른 사람이 그 코드를 이해할 수 있는가?
